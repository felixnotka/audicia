# AudiciaReport & AudiciaPolicy (Example Output)

Example AudiciaReport and AudiciaPolicy as generated by the operator. These are
read-only references – you do **not** create these. The operator generates and
updates them automatically. The AudiciaReport contains observed rules and
compliance data, while the AudiciaPolicy holds the suggested RBAC manifests.

**See also:** [AudiciaReport CRD](../reference/crd-audiciareport.md) |
[AudiciaPolicy CRD](../reference/crd-audiciapolicy.md) |
[Compliance Scoring](../concepts/compliance-scoring.md)

## AudiciaPolicy

```yaml
apiVersion: audicia.io/v1alpha1
kind: AudiciaPolicy
metadata:
  name: policy-sa-backend
  namespace: my-team
  labels:
    audicia.io/subject-kind: ServiceAccount
    audicia.io/subject-name: backend
    audicia.io/subject-namespace: my-team
spec:
  subject:
    kind: ServiceAccount
    name: backend
    namespace: my-team
  sourceRef: "dev-cluster-audit"
  manifests:
    - |
        apiVersion: rbac.authorization.k8s.io/v1
        kind: Role
        metadata:
          name: suggested-backend-role
          namespace: my-team
        rules:
          - apiGroups: [""]
            resources: ["pods"]
            verbs: ["get", "list", "watch"]
          - apiGroups: [""]
            resources: ["pods/exec"]
            verbs: ["create"]
          - apiGroups: [""]
            resources: ["configmaps"]
            verbs: ["get", "watch"]
          - apiGroups: ["apps"]
            resources: ["deployments"]
            verbs: ["get"]
    - |
        apiVersion: rbac.authorization.k8s.io/v1
        kind: RoleBinding
        metadata:
          name: suggested-backend-binding
          namespace: my-team
        roleRef:
          apiGroup: rbac.authorization.k8s.io
          kind: Role
          name: suggested-backend-role
        subjects:
          - kind: ServiceAccount
            name: backend
            namespace: my-team

status:
  state: Pending
  ruleCount: 4
  conditions:
    - type: Ready
      status: "True"
      lastTransitionTime: "2026-02-14T12:05:00Z"
      message: "Policy generated successfully"
```

## Reading These Resources

- **AudiciaReport `observedRules`** – The 4 distinct API access patterns this
  ServiceAccount uses. Each includes timestamps and call counts.
- **AudiciaReport `compliance.score: 50`** – This SA uses only 4 of its 8
  effective permissions (Yellow). The 4 unused grants include access to
  `secrets`.
- **AudiciaReport `eventsProcessed: 24458`** – Total audit events that
  contributed to this report.
- **AudiciaPolicy `spec.manifests`** – A ready-to-apply Role and RoleBinding
  granting exactly the observed permissions.
- **AudiciaPolicy `status.state: Pending`** – The policy awaits review and
  approval before being applied.
- **AudiciaPolicy `spec.sourceRef`** – Links back to the AudiciaSource that
  produced this policy.
